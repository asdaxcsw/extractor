# 解决GitHub推送失败问题

## 问题诊断

你遇到的错误：`fatal: unable to access 'https://github.com/asdaxcsw/extractor.git/': Recv failure: Connection was reset`

这是网络连接问题，可能的原因：
1. 网络不稳定
2. 防火墙或代理设置
3. GitHub服务暂时不可用
4. DNS解析问题

## 解决方案

### 方案1：重试推送（最简单）

有时只是临时网络问题，直接重试即可：

```powershell
cd C:\Users\Administrator\.cursor\projects\empty-window
git push -u origin main
```

### 方案2：配置Git代理（如果你使用代理）

如果你使用VPN或代理上网：

```powershell
# 设置HTTP代理（替换为你的代理地址和端口）
git config --global http.proxy http://127.0.0.1:7890
git config --global https.proxy http://127.0.0.1:7890

# 然后重试推送
git push -u origin main

# 如果不需要代理，取消设置
git config --global --unset http.proxy
git config --global --unset https.proxy
```

### 方案3：使用SSH代替HTTPS

SSH连接通常更稳定：

1. **生成SSH密钥**（如果还没有）：

```powershell
ssh-keygen -t ed25519 -C "你的邮箱@example.com"
```

按Enter使用默认路径，可以设置密码或直接按Enter跳过。

2. **复制公钥**：

```powershell
cat ~/.ssh/id_ed25519.pub
```

复制输出的内容。

3. **添加到GitHub**：
   - 访问 https://github.com/settings/keys
   - 点击 "New SSH key"
   - 粘贴公钥内容
   - 点击 "Add SSH key"

4. **更改远程仓库地址为SSH**：

```powershell
cd C:\Users\Administrator\.cursor\projects\empty-window
git remote set-url origin git@github.com:asdaxcsw/extractor.git
git push -u origin main
```

### 方案4：增加Git超时时间

```powershell
git config --global http.postBuffer 524288000
git config --global http.lowSpeedLimit 0
git config --global http.lowSpeedTime 999999
git push -u origin main
```

### 方案5：检查GitHub状态

访问 https://www.githubstatus.com/ 查看GitHub服务是否正常。

### 方案6：使用GitHub Desktop（图形界面）

如果命令行一直失败，可以使用GitHub Desktop：

1. 下载：https://desktop.github.com/
2. 安装并登录GitHub账号
3. 点击 "Add" > "Add existing repository"
4. 选择 `C:\Users\Administrator\.cursor\projects\empty-window`
5. 点击 "Publish repository" 推送到GitHub

## 快速诊断脚本

运行以下命令检查网络连接：

```powershell
# 测试GitHub连接
ping github.com

# 测试HTTPS连接
curl -I https://github.com

# 查看当前Git配置
git config --list
```

## 推荐步骤

1. **先重试一次**（可能只是临时网络问题）：
   ```powershell
   git push -u origin main
   ```

2. **如果还是失败，使用SSH方式**（更稳定）：
   - 生成SSH密钥
   - 添加到GitHub
   - 更改远程地址为SSH
   - 重新推送

3. **如果都不行，使用GitHub Desktop**（最简单）

## 验证推送成功

推送成功后，访问你的仓库页面：
https://github.com/asdaxcsw/extractor

你应该能看到所有文件，并且在 "Actions" 标签页看到自动构建正在运行。
