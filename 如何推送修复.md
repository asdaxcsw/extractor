# 如何推送修复到GitHub

我已经修复了GitHub Actions配置文件中的错误（将 actions/upload-artifact 从 v3 更新到 v4）。

现在需要将修复推送到GitHub。

## 使用GitHub Desktop推送（推荐）

1. **打开GitHub Desktop**

2. **查看更改**
   - 你会看到 `.github/workflows/build.yml` 文件已被修改
   - 左侧显示文件列表，右侧显示具体更改

3. **提交更改**
   - 在左下角的 "Summary" 框中输入：`Fix: Update actions/upload-artifact to v4`
   - 点击蓝色的 "Commit to main" 按钮

4. **推送到GitHub**
   - 点击顶部的 "Push origin" 按钮
   - 等待推送完成

5. **查看自动构建**
   - 推送成功后，访问：https://github.com/asdaxcsw/extractor/actions
   - 你会看到新的构建任务自动开始
   - 这次应该会成功构建

## 或者使用命令行（如果网络稳定）

打开PowerShell，运行：

```powershell
cd C:\Users\Administrator\.cursor\projects\empty-window
git add .github/workflows/build.yml
git commit -m "Fix: Update actions/upload-artifact to v4"
git push
```

## 构建完成后

1. 访问 https://github.com/asdaxcsw/extractor/actions
2. 等待构建完成（15-30分钟）
3. 点击构建记录
4. 在 "Artifacts" 部分下载 "android-apk"
5. 解压ZIP文件获取APK
6. 安装到Android设备

---

推荐使用GitHub Desktop，因为它处理网络问题更好。
